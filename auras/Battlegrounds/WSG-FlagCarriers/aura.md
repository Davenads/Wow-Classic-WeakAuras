# WSG Flag Carriers

Shows the name of the player currently carrying each flag in Warsong Gulch, at top-center
of the screen next to the capture scores. The Alliance player holding the Horde flag shows
on the left in blue; the Horde player holding the Alliance flag shows on the right in red.
Names appear only while a flag is actually being carried.

## Status

The `export.txt` `!WA:2!` string is **machine-generated** (see "How the string was built")
and passes a lossless encode/decode round-trip. It has **not yet been imported/tested
in-game** — treat it as import-ready, not confirmed working, until verified in a WSG match.

## Purpose

In Warsong Gulch each team scores by carrying the **enemy** flag to their base. You can
only scan auras on your own team, so the enemy carrier is invisible to `UnitAura` across
the map. The game instead announces every flag pickup/drop/return/capture to everyone via
`CHAT_MSG_BG_SYSTEM_*` system messages (arg1 = the text). This aura parses those messages
and displays both carriers' names.

Left/right pairs each name with its team's score: a team's score increments from capturing
the enemy flag its own player carries, so the **Alliance** score (left) pairs with the
Alliance player holding the **Horde** flag, and the **Horde** score (right) pairs with the
Horde player holding the **Alliance** flag.

## Import

1. In-game: `/wa` → **Import**, paste the contents of `export.txt`.
2. It loads only in Warsong Gulch (Load → Zone = Warsong Gulch). Join WSG to see it.
3. Nudge position if needed: it anchors to screen TOP with a small offset; drag/adjust so
   it sits right by the `0/3 · 0/3` counters.

## How it works (code)

Single `text` region with one **custom Status trigger** and a **`%c` custom-text** renderer.

- `code/init.lua` → **Actions → On Init**: creates `aura_env.fc = {}`.
- `code/trigger.lua` → **Trigger 1 → Custom → Status** (Check On: Event). Events box:
  `CHAT_MSG_BG_SYSTEM_ALLIANCE, CHAT_MSG_BG_SYSTEM_HORDE, CHAT_MSG_BG_SYSTEM_NEUTRAL, PLAYER_ENTERING_WORLD`.
  Parses the messages into `aura_env.fc.alliance` / `aura_env.fc.horde` and returns whether
  either flag is carried (show/hide).
- `code/custom_text.lua` → **Display → Text → `%c`** (Display Text is `%c`). Renders the two
  colored names.

These files are the source of truth for the logic; `export.txt` is regenerated from them
(they are kept in sync — see below).

## How the string was built

Rather than author a WA table blind, the string was generated by:
1. Fetching a real, valid WSG-carrier text aura from wago and decoding it with
   `tools/` (`node-weakauras-parser`) to get a complete, correct field set.
2. Replacing its identity, triggers, actions, text, and load with this aura's `code/*.lua`
   (read verbatim from the files, so `code/` and `export.txt` cannot drift).
3. Encoding back to `!WA:2!` and round-trip-verifying the decode.

`aura.json` is the decoded envelope, committed for readable diffs. To regenerate after
editing `code/*.lua`, re-run the build (the one-off builder lives in the session scratch;
ask to re-materialize it, or rebuild via `tools/` following these steps).

## Testing notes (verify in-game)

- On "The Horde Flag was picked up by X!" a blue name appears (left); on drop/return it
  clears; on a capture both clear. Repeat for the Alliance flag (red, right).
- Locale: patterns in `code/trigger.lua` are enUS — the only locale-specific part.
- Known limitation: a carrier who grabbed the flag **before** you zoned in won't be known
  until the next flag event. Optional enhancement: also scan your own raid for the flag
  aura to recover your team's carrier on join.
- Positional upgrade (optional): the score widgets live in `UIWidgetTopCenterContainerFrame`
  (children with tooltip "Alliance flag captures" / "Horde flag captures"). A future version
  could anchor the names directly to those widgets to sit exactly beside each counter.
